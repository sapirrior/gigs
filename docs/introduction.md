# Introduction to GIGs

General Interactive Guides (GIGs) are comprehensive technical reference pages rendered by the `gig` engine. 

## The GIG Philosophy

A GIG is not a "cheat sheet," a "help file," or a "summary." It is a **full-depth manual page** designed for the modern terminal. 

The goal of the GIG project is to provide a platform where manual pages are:
- **Easier to Write**: Using the structured and semantic `.gg` syntax.
- **Easier to Read**: Using a modern, interactive, and visually clear interface.

### What's in a Name?
We call these pages **GIG** (singular) and **GIGs** (plural). Because while traditional systems have "man" pages, we figured that reading documentation shouldn't feel like a lonely date with a 1970s mainframe. A "GIG" is a performance of technical data, and "GIGs" are the world tour. (Note: Stage diving into your terminal is not recommended). 😭😛

### Informative Depth & Structure
While the interface is modern, the rigor is traditional. A GIG must be **as informative as a manual page**, ensuring that the information provided is complete and reliable. 

To maintain a professional and predictable experience, all GIGs must preserve the fundamental pillars of a reference manual: `NAME`, `SYNOPSIS`, and `DESCRIPTION`. Within this framework, contributors are free to add new sections, headers, or modern context—as long as the page remains consistent with the rest of the library and doesn't diverge into a "fully different" structure.
