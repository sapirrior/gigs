# How to Write General Interactive Guides (GIGs)

`.gg` files are rendered by the `gig` engine to provide a structured, interactive, and visually enhanced manual page experience.

## Emulating Manual Pages

The goal of a `gig` page is to leverage powerful syntax while adhering to the familiar structure and style of traditional Unix manual pages.

### Core Principles

- **Clear Sections** : Use logical blocks like `NAME`, `SYNOPSIS`, and `DESCRIPTION`.
- **Concise Language** : Maintain the brevity and precision expected of technical documentation.
- **Structured Hierarchy** : Use block types and inline styling to present info clearly.
- **Consistency** : Ensure a cohesive experience across all guides in the collection.
